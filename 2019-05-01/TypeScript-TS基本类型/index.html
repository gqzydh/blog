TypeScript 基本类型

2019_05_01 |
TypeScript
| TypeScript

在 TS 中，用 let 的变量需要指定类型

boolean 类型

Number 类型

支持十六进制、十进制、八进制和二进制；

String 类型

也可以用单引号或双引号去包裹 string 数据

如果字符串和变量拼接或者换行的话可以使用 '（反引号）来包裹整个数据，内部变量通过 ${ expr }方式导入

相当于：

Array 类型:

两种定义方式：

  1. 基本数据类型加上 [] 定义数组
  2. 数组类型加<基本类型>定义数组

Any 类型：（任何）

没有强制限定哪种类型，随时切换类型都可以

相比于对象，也可调用其中的方法

对象类型的变量只允许你给它赋值，不能调用它的任意方法，即使这个方法是存在的

当你知道部分类型但是不全的时候也可使用 any 类型

Void 类型：

有点和 any 类型相反，意思为没有任何类型；
通常可视为不返回值的返回类型

不适合用来声明变量，因为只能用来声明 undefined 和 null；

Null and Undefined 类型：

事实上 null 和 undefined 有它们各自的类型名，就是 null 和 undefined
他们只能用来声明自己

  • 默认的 null 和 undefined 是其他类型的子类型
  • 意味着你可以将 null 和 undefined 赋值给其他所有类型如 number
  • 然而，当使用 – strictNullChecks 标记的时候，它们只能被分配给 void 和它们自己的类型
  • 这对验证很多公共错误很有帮助
  • 当你要通过 string、null 和 undefined 中任一个的验证时，可用联合类型 string | null | undefined 来校验

Tuple 类型：（数组）

允许你表达一个固定数量的元素类型是已知的数组，比如你想表达一对字符串和数字的值；
意思就是初始化数组的时候前面的数据必须和指定的元素类型和顺序一致，超出指定数量的只限定类型，不限定顺序；

Enum 类型：（列举，枚举）

枚举是一种更友好的名称数值集
默认的，枚举里面分子的开始数字是0，你可以手动的设置第一个值得数字来改变其开始数字；

也可以设置所有你想自己设置的值对应的数字;
没设置的值对应数字会从前面一个值递增

甚至可以通过数字值去获取枚举中的值

Never 类型

表示用不存在值得类型

  • 例如：是那些总是抛出异常或不会有返回值的函数表达式或箭头函数表达式的返回类型
  • 变量也可能是 never 类型，当它们被永不为真的类型所约束时
    是任何类型的子类型，也可以赋值给任何类型
    没有类型是 never 类型的子类型，除了 never 类型外没有类型可以赋值给 never 类型（包括 any）

assertions 类型：（断言）

  • 有时候会出现一种情况，你会比 TypeScript 更了解某个值得信息
    这通常会发生在你知道一个实体具有比它现有类型更确切的类型
  • 断言类型是一种告诉编译器「相信我，我知道我在做什么」的方法
  • 断言类型就好比其他语言的类型转换，但是不进行特殊的数据校验和重组
  • 它没有运行时的影响，只在编译阶段起作用，TypeScript 会假设你已经进行了必须的检查

断言类型有两种形式：

一种是「尖括号」语法

另外一种是 as 语法

两种形式是等价的。 至于使用哪个大多数情况下是凭个人喜好；然而，当你在 TypeScript 里使用 JSX 时，只有 as 语法断言是被允许的。
