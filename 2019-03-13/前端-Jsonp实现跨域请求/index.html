前端性能优化
如今浏览器能够实现的特性越来越多，并且网络逐渐向移动设备转移，使我们的前端代码更加紧凑，如何优化，就变得越来越重要了。

原则

  • 减少 CPU 计算， 减少网络请求
  • 多使用内存，缓存或者其他方法

加载资源优化

  1. 静态资源css,js文件压缩合并；
  2. 小图片，使用雪碧图；
  3. 静态资源缓存；
  4. 使用CDN让资源加载更快；
  5. 使用SSR 后端渲染，数据直接输出到HTML中；

渲染优化

  1. CSS 放前面，js放后面；
  2. 懒加载（lazyload, 图片懒加载，下拉加载更多）；
  3. 减少DOM查询，对DOM查询做缓存；
  4. 减少DOM操作，多个操作尽量合并在一起执行；
  5. 事件节流
  6. 尽早执行操作（如 DOMContentLoaded)；

减少 HTTP 请求数量

在浏览器与服务器进行通信时，主要是通过 HTTP 进行通信。 浏览器与服务器需要经过三次通信，每次通信需要花费大量时间。
而且不同浏览器对资源文件并发请求数量有限（不同浏览器允许并发数），一旦 HTTP 请求数量达到一定数量，资源请求就存在等待状态，因此减少 HTTP 的请求数量可以很大程度上对网站性能进行优化。

性能优化的方法

（1） 减少http请求次数：CSS Sprites, JS、CSS源码压缩、图片大小控制合适；网页Gzip，CDN托管，data缓存 ，图片服务器。

（2） 前端模板 JS+数据，减少由于HTML标签导致的带宽浪费，前端用变量保存AJAX请求结果，每次操作本地变量，不用请求，减少请求次数

（3） 用innerHTML代替DOM操作，减少DOM操作次数，优化javascript性能。

（4） 当需要设置的样式很多时设置className而不是直接操作style。

（5） 少用全局变量、缓存DOM节点查找的结果。减少IO读取操作。

（6） 避免使用CSS Expression（css表达式)又称Dynamic properties(动态属性)。

（7） 图片预加载，将样式表放在顶部，将脚本放在底部 加上时间戳。
