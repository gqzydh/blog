【前端】Web前端性能优化

2017_10_17 |
前端
| 前端

前端性能优化点

涉及到的层面：

  • 网络层面
  • 构建层面
  • 浏览器渲染层面
  • 服务器层面

涉及的功能

  • 资源的合并与压缩
  • 图片编解码原理和类型选择
  • 浏览器渲染机制

资源的合并与压缩

减少http请求数量和减少请求资源大小
html, css, js, 文件合并， 开启gzip

实战优化点

  1. 用translate替代top改变

  2. 用opacity替代visibility

    即使不可见的元素也会占据页面上的空间

  3. 不要一条一条地修改 DOM 的样式，预先定义好 class，然后修改 DOM 的 className
  4. 把 DOM 离线后修改，比如：先把 DOM 给 display:none (有一次 Reflow)，然后你修改100次，然后再把它显示出来
  5. 不要把 DOM 结点的属性值放在一个循环里当成循环里的变量
  6. 不要使用 table 布局，可能很小的一个小改动会造成整个 table 的重新布局
  7. 动画实现的速度的选择
  8. 对于动画新建图层
  9. 启用 GPU 硬件加速
